## 背景

近年来，慢性病已成为全球公共卫生的主要负担，非传染性慢性疾病约占全球死亡人数的 74%。针对慢性病患者的健康素养研究表明，仅约 21.6% 的患者具备充分的健康素养，而健康素养不足与药物理解准确性显著相关，这直接影响患者对病历和用药指示的理解和执行效果。此外，临床沟通研究指出，患者离开诊室后常无法准确回忆医生解释的内容，对健康行为形成负面影响。在市场规模方面，全球慢性病管理市场预计将从约 60 亿美元增长至逾 170 亿美元，显示出慢性病管理领域需求的持续扩大。

## 产品定位

本项目构建了一个面向患者的医疗解析与疾病管理智能体，解决用户在就诊后阶段遇到的医学信息理解困难与长期疾病管理问题。

## 用户画像

### 核心用户

慢性病或长期随访患者，典型特征包括：
 
- 需要长期反复检查与持续用药  
- 拥有多份病历、检查报告或影像说明  
- 缺乏医学背景，难以理解专业医疗文本  
- 需要对比不同时间段的病情变化

| 用户特征 | 描述 |
|--------|------|
| 健康信息复杂 | 多病共存、长期用药、指标众多 |
| 医疗文档繁杂 | 多次病历、检查报告、影像记录 |
| 信息理解门槛高 | 医学术语多、解释不足 |
| 管理周期长 | 需要长期记录、对比与提醒 |

### 次级用户

- 陪诊者（如家属、护工）
- 长期健康管理需求人群

## 痛点

### 1. 医疗文本不可读

- 病历、检查单和影像报告充斥医学术语与缩写  
- 普通患者难以直接理解其含义

### 2. 医疗信息高度碎片化

- 病历分散在不同时间、不同医院和系统  
- 难以进行长期对比与整体回顾

### 3. 就诊后阶段缺乏支持

- 就诊交流时间有限  
- 回家后对病历内容、医嘱细节记忆模糊  

### 4. 执行与管理困难

- 因理解不足导致用药或复诊执行不稳定  
- 无法形成清晰、连续的个人健康记录

---

## 需求分析

| 需求类型 | 具体需求 |
|--------|----------|
| 信息理解 | 将专业医学文本转化为通俗易懂的内容 |
| 信息结构化 | 提取诊断、症状、用药、风险等关键字段 |
| 历史对比 | 支持多次就诊记录的纵向比较 |
| 长期记忆 | 将关键信息长期存储、可随时回顾 |
| 行动支持 | 提供用药与复诊等提醒机制 |

## 功能与使用场景

### 1️ 病历与影像解析

用户上传病历或影像报告：

- 利用 MedGemma 进行医学语义理解  
- 提取诊断、用药、关键指标等信息
- 生成结构化摘要与可读解释

**适用场景：**

- 门诊或住院病历解读  
- 出院记录回顾  
- 检查报告理解

---

### 2️ 医学知识补充

结合 Web Search 能力：

- 对医学术语提供权威解释  
- 补充与病历相关的医学背景信息  
- 在不越界诊断的前提下增强患者理解

---

### 3 用药提醒与长期管理辅助

- 生成用药提醒  
- 提示复诊或随访时间
